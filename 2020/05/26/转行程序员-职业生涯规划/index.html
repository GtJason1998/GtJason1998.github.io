该篇博客参考自以下视频，总结提炼并引入了一部分自己的思考：

对专业不感兴趣？转行当程序员？这些致命问题你想过吗？微软码农向你丢来灵魂三问！

1. 思考自己选择某一工作方向的动机

2. 学会对公司做背调

  • 公司层面
    • 公司的部门团队
    • 公司的业务，这个公司靠什么赚钱
    • 公司对应的业务有什么竞品
    • 公司的行业壁垒
    • 公司的价值观，公司的管理层是什么样的人
  • 研发层面
    • 公司内部工程师文化如何
      • code review
      • 技术分享会
      • 工程组织架构是否合理
      • 公司开源文化，公司Github开源项目、commit、issues等
    • 调查公司业务的法律风险
      • P2P暴雷
      • 非法爬虫
    • 裁员风险
      • 公司产品业务线单一，公司经营容错率低，可能会有比较大的裁员风险

3. 技术岗位分类

  • 测试
  • 后端
  • 前端
  • 客户端(IOS/Android)
  • 算法岗
    • 算法岗也属于后端的一部分，后端中针对数据的一些特殊业务(比如搜索、推荐、反垃圾、异常检测、图像、NLP、Speech)属于算法岗

4. 关于各个岗位需要了解什么

  • 成长路线
    • 学会使用英文搜索，Bing\Google搜索引擎，比如关于后端工程师搜索backend engineer career path、roadmap等关键词，英文互联网环境较中文互联网环境更靠谱
    • 对照成长路线，少说多做，并不停反思自己的选择动机是否还成立
  • 发展技术方面，需注重术的部分
    • 技，主要指框架
    • 术，即理论基础

      框架可能快速更新迭代，但理论基础不会

5. 学会主动联系公司hr和公司员工，询问公司人才岗位需求
